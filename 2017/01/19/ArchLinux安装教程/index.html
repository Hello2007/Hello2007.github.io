Archlinux安装教程

发表于 2017-01-19 | 分类于 随笔

写在前面

Archlinux官方有着完整的中文Wiki，因此，这种教程大概是没什么必要的，但是如果你只想按照步骤一步步装完，而不想跟着官方链接跳来跳去的话，那么这篇文章还有那么点存在的价值。

教程开始

官方中文Wiki链接:点此跳转)
前期从分区到安装官方都有详尽的说明及命令，不再赘述。由于官方从安装引导程序处开始放各种链接，因此本教程从此处开始。
另外，安装过程中可能由于地区问题影响下载速度，因此，这里建议使用清华、浙大的源：

执行以下命令，用Vim来编辑/etc/pacman.d/mirrorlist这个文件

vim /etc/pacman.d/mirrorlist

找到标有 China 的的镜像源，添加两条记录：

Server = http://mirrors.tuna.tsinghua.edu.cn/archlinux/$repo/os/$arch
Server = http://mirrors.zju.edu.cn/archlinux/$repo/os/$arch

安装引导程序

如果是Intel CPU，在终端执行下列命令

pacman -S intel-ucode

安装Bootloader，这里我们安装最流行的Grub2。首先安装os-prober这个包，它可以配合Grub检测已经存在的系统，自动设置启动选项。

pacman -S os-prober

如果为BIOS/MBR引导方式：

  1. 安装grub包：

    pacman -S grub

  2. 部署grub：

    grub-install –target=i386-pc /dev/sdx

注意这里的sdx应该为硬盘（例如/dev/sda），而不是形如/dev/sda1这样的分区。

  1. 生成配置文件：

    grub-mkconfig -o /boot/grub/grub.cfg

如果报warning failed to connect to lvmetad，falling back to device scanning.错误。简单的方法是编辑/etc/lvm/lvm.conf这个文件，找到use_lvmetad = 1将1修改为0，保存，重新配置grub。

如果为EFI/GPT引导方式：

  1. 安装grub与efibootmgr两个包：

    pacman -S grub efibootmgr

  2. 部署grub：

    grub-install –target=x86_64-efi –efi-directory=/boot –bootloader-id=grub

  3. 生成配置文件：

    grub-mkconfig -o /boot/grub/grub.cfg

如果报warning failed to connect to lvmetad，falling back to device scanning.错误。简单的方法是编辑/etc/lvm/lvm.conf这个文件，找到use_lvmetad = 1将1修改为0，保存，重新配置grub。

安装后检查

强烈建议使用如下命令检查是否成功生成各系统的入口，如果没有正常生成会出现开机没有系统入口的情况：

vim /boot/grub/grub.cfg

检查接近末尾的menuentry部分是否有Archlinux入口，若不存在，请先检查/boot目录是否正确部署linux内核：

cd /boot
ls

查看是否有initramfs-linux-fallback.img initramfs-linux.img intel-ucode.img vmlinuz-linux这几个文件，如果都没有，说明linux内核没有被正确部署，很有可能是/boot目录没有被正确挂载导致的，确认/boot目录无误后，可以重新部署linux内核：

pacman -S linux

再重新生成配置文件，就可以找到系统入口。

重启

接下来，你需要进行重启来启动已经安装好的系统，执行如下命令：

exit
reboot
